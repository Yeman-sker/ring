# Ring PRD — 功能规格

版本：v1.0-draft
日期：2026-07-05
状态：MVP 定义完成，待评审

---

## 1. 产品定位

Ring 是一套**纯静态个人博客系统**，目标是：

- **激励写作习惯**：通过视觉反馈（写作日历）让写作变成可感知的持续行为
- **记录个人成长**：按时间线回溯，能看到自己的积累
- **极简运维**：零后端，内容即 Markdown，部署即 HTML

不是一个通用博客框架，不是给别人看的作品集，是一个「写给自己的成长日志」。

---

## 2. 用户画像

**唯一用户：博客所有者（Boss 本人）**

- 用 Markdown 写文章，通过 git push 发布
- 需要视觉反馈来保持写作频率
- 偶尔回头看自己的成长轨迹
- 对花里胡哨的功能无感，但要求架构能支撑以后加功能

## 3. 功能需求

### 3.1 P0 — MVP 必备

#### F1: 文章发布
- **描述**：在 `src/content/posts/` 下创建 `.md` 或 `.mdx` 文件即发布
- **元数据**：title / date / category / tags / summary / draft
- **正文**：标准 Markdown，支持代码高亮、图片、链接
- **draft 机制**：`draft: true` 的文章构建时不输出，本地 dev 可见

#### F2: 文章列表（首页）
- **描述**：按时间倒序展示已发布文章
- **每篇文章卡片**：标题、日期、分类标签、摘要
- **分页**：每页 10 篇，底部页码或「加载更多」
- **空状态**：没有文章时显示一句引导语

#### F3: 文章详情
- **描述**：单篇文章的完整阅读视图
- **显示**：标题、日期、分类标签、正文（Markdown 渲染）、上下篇导航
- **SEO**：自动生成 `<title>` 和 `<meta description>`（取 summary 字段）
- **未来预留**：阅读时长估算、目录（TOC）

#### F4: 写作日历
- **描述**：首页或独立页面的类 GitHub contribution 日历
- **展示**：365 天或自定义时间范围的格子图
- **颜色深度**：当天文章数越多颜色越深
- **交互**：hover 显示日期 + 文章数，click 跳转到当天文章列表
- **实现**：客户端 Island 组件，纯前端计算

#### F5: 时间线 / 归档
- **描述**：按年-月层级展示所有文章
- **路径**：`/archive/`
- **交互**：侧边年份列表，点击展开该年各月文章
- **空月份不显示**

### 3.2 P1 — 有更好、没有也能用

#### F6: 分类浏览
- **描述**：按 category 字段筛选文章
- **路径**：`/category/<name>/`
- **分类列表页**：展示所有分类及其文章数量

#### F7: 标签系统
- **描述**：每篇文章可打多个 tag
- **路径**：`/tag/<name>/`
- **标签云**：首页侧栏（可选）

#### F8: RSS 订阅
- **路径**：`/rss.xml`
- **内容**：最近 20 篇文章全文或摘要
- **标准**：Atom 1.0 或 RSS 2.0

#### F9: 关于页
- **描述**：独立页面，自我介绍
- **路径**：`/about/`
- **内容**：Markdown 或 Astro 组件

### 3.3 P2 — 后续版本

| 功能 | 触发条件 |
|------|---------|
| 全文搜索 | 文章超过 30 篇 |
| 暗色模式 | Boss 提出来 |
| 系列/专题 | 同一主题连续写 5+ 篇 |
| 评论区（Giscus 等） | Boss 需要反馈 |
| 文章阅读统计 | 好奇心驱动 |
| 邮件 Newsletter | 需要分发 |
| OG 图片自动生成 | 想分享到社交平台 |
| AI 辅助写作 / 润色 | 懒了 |

## 4. 非功能需求

### N1: 性能
- 首页加载 < 2s（含日历组件）
- 构建时间 < 30s（500 篇文章以内）
- Lighthouse 评分 ≥ 95

### N2: 可扩展性
- 新增内容类型（如 notes、says）只需加 Collection 定义 + 路由
- 新增前端功能加 Island 组件，不影响既有静态页面
- Content Schema 用 Zod，新增字段对旧文章无感

### N3: 运维
- 部署：Vercel CLI 或 GitHub Actions 自动部署
- 发布流程：`git add` → `git commit` → `git push`
- 无需数据库备份、无需服务端监控

### N4: 可访问性
- 语义化 HTML
- 图片有 alt 文本
- 键盘可导航

---

## 5. 不做的事

- ❌ 管理后台（CMS）—— 写 Markdown + git push 就是管理
- ❌ 用户登录/注册 —— 只有自己，不需要
- ❌ 评论系统 —— V1 不需要，未来可以用 Giscus
- ❌ 多语言 —— Boss 只写中文
- ❌ 第三方 API 集成 —— 不接网易云、B 站、GitHub 仪表盘那些
- ❌ 社交分享按钮 —— 不加
- ❌ 埋点分析 —— 不自查
